# Release Evidence Directory

Each current release run must produce:

- `reports/releases/<tag>/manifest.json`
- `reports/releases/<tag>/checksums.sha256`
- `reports/releases/<tag>/rollback/db-rollback-readiness.json`
- `reports/releases/<tag>/rollback/drill.json`
- `reports/releases/<tag>/canary/canary-rollout-evidence.json`
- `reports/releases/<tag>/canary/canary-rollout-dryrun.log`

Validation rules:

1. `db-rollback-readiness.json` must be generated by `scripts/release/verify_db_rollback_readiness.py`.
2. `summary.missing_policy` must be `0`.
3. `summary.blocked_without_down` must be `0`.
4. `drill.json` must include required keys:
   - `release_tag`
   - `executed_at`
   - `executor`
   - `strategy`
   - `result`
   - `migrations_checked`
5. Canary evidence can be generated with:
   - `scripts/release/capture_canary_rollout_evidence.sh [release_tag]`

Canonical evidence rules:

- `reports/releases/<tag>/` is an evidence directory, not the release verdict by itself.
- Only the **current run** bundle generated by `release-evidence-attest.yml` is canonical for attestation and release proof.
- Historical sample files under this directory are documentation examples and may be dirty or incomplete.
