# Support

## What This Repository Supports

This repository supports:

- source-first local development,
- repo-side governance and CI parity work,
- reproducible bug reports against documented commands,
- limited public issue triage.

This repository does **not** promise:

- hosted service uptime,
- production support,
- guaranteed provider quotas or third-party API availability,
- customized onboarding for private environments,
- long-term maintenance for every optional external lane.

## Best Place To Start

- Public/source-first onboarding: `README.md`, `docs/start-here.md`
- Local operator runbook: `docs/runbook-local.md`
- Repo-side vs external completion model: `docs/reference/done-model.md`

## How To Ask For Help

Open an issue when:

- a documented source-first command is broken,
- a repo-side gate result contradicts the documented contract,
- a public artifact or sample appears unsafe or mislabeled.

Use the security process in `SECURITY.md` instead of public issues for sensitive reports.
